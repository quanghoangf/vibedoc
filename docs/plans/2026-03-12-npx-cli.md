# npx vibedoc CLI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Package VibeDoc so `npx vibedoc` starts the app (Next.js + WebSocket server) from any project directory.

**Architecture:** A Node.js bin script sets `VIBEDOC_ROOT=cwd()`, then spawns `next start` and `ws-server.js` as child processes from the package install directory. The pre-built `.next/` bundle is included in the npm package so no build step is needed at runtime.

**Tech Stack:** Node.js child_process, Next.js production server, Yjs WebSocket server (`ws`)

---

### Task 1: Create the bin script

**Files:**
- Create: `bin/vibedoc.js`

**Step 1: Create `bin/` directory and write the script**

```js
#!/usr/bin/env node
'use strict';

const { spawn } = require('child_process');
const path = require('path');

const PKG_DIR = path.resolve(__dirname, '..');
const VIBEDOC_ROOT = process.env.VIBEDOC_ROOT || process.cwd();
const PORT = process.env.PORT || '3000';
const WS_PORT = process.env.WS_PORT || '1234';

console.log(`[vibedoc] Starting...`);
console.log(`[vibedoc] Project root: ${VIBEDOC_ROOT}`);
console.log(`[vibedoc] App: http://localhost:${PORT}`);
console.log(`[vibedoc] WS:  ws://localhost:${WS_PORT}`);

const env = { ...process.env, VIBEDOC_ROOT };

const next = spawn(
  process.execPath,
  [path.join(PKG_DIR, 'node_modules/.bin/next'), 'start', '--port', PORT],
  { cwd: PKG_DIR, stdio: 'inherit', env }
);

const ws = spawn(
  process.execPath,
  [path.join(PKG_DIR, 'ws-server.js')],
  { cwd: PKG_DIR, stdio: 'inherit', env }
);

function shutdown() {
  next.kill();
  ws.kill();
}

process.on('SIGINT', shutdown);
process.on('SIGTERM', shutdown);

next.on('exit', code => process.exit(code ?? 0));
ws.on('exit', () => {});
```

**Step 2: Make it executable**

```bash
chmod +x bin/vibedoc.js
```

**Step 3: Verify it exists**

```bash
ls -la bin/vibedoc.js
```
Expected: `-rwxr-xr-x ... bin/vibedoc.js`

**Step 4: Commit**

```bash
git add bin/vibedoc.js
git commit -m "feat: add bin/vibedoc.js CLI entry point"
```

### Task 2: Update package.json

**Files:**
- Modify: `package.json`

**Step 1: Apply all changes at once**

Update `package.json` with these changes:
- Remove `"private": true`
- Add `"bin": { "vibedoc": "./bin/vibedoc.js" }`
- Add `"files"` array
- Move `ws` from devDependencies to dependencies (check if `ws` is listed; it may be a transitive dep — check with `cat package.json | grep '"ws"'`)

The final relevant fields should look like:

```json
{
  "name": "vibedoc",
  "version": "1.0.0",
  "description": "Local-first project intelligence for AI-assisted development. Web UI + MCP server in one.",
  "bin": {
    "vibedoc": "./bin/vibedoc.js"
  },
  "files": [
    "bin/",
    ".next/",
    "public/",
    "next.config.js",
    "ws-server.js",
    "package.json"
  ],
  ...
}
```

Note: `ws` is a dependency of `y-websocket` so it will already be in `node_modules` when installed. No need to add it explicitly unless `ws-server.js` fails to resolve it.

Expected: `valid`

**Step 3: Commit**

Expected output:
```

[vibedoc] Project root: /tmp/vibedoc-test
[vibedoc] App: http://localhost:3000
[vibedoc] WS:  ws://localhost:1234
```
Then open http://localhost:3000 in the browser — the app should load with `/tmp/vibedoc-test` as the project root (empty state is fine).

**Step 4: Stop with Ctrl+C — both processes should exit cleanly**

**Step 5: Commit build output note**

Add `.next/` to git if not already tracked (check `.gitignore` first):

```bash
cat .gitignore | grep next
```

If `.next` is in `.gitignore`, you need to either:
- Remove it from `.gitignore` (so it's committed and included in npm publish), OR
- Add a `.npmignore` that does NOT exclude `.next/` (npm uses `.npmignore` over `.gitignore` when present)

Recommended: create `.npmignore` that includes everything needed and excludes source:

```
# .npmignore — what to EXCLUDE from npm package
src/
docs/
plans/
memory/
*.md
*.ts
tsconfig.json
.env*
pnpm-lock.yaml
```

Then `.next/` stays in `.gitignore` (no build artifacts in git) but ships in the npm package.

```bash
git add .npmignore
git commit -m "feat: add .npmignore to include .next/ in npm package"
```
